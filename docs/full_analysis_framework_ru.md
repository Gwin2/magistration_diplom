# Полный аналитический каркас исследования

## 1. Исследовательские вопросы

1. Как меняется точность детекции БПЛА при ухудшении погодных/визуальных условий?
2. Какие архитектуры дают лучший баланс точности и скорости для практического применения?
3. Насколько устойчивы модели к маневрам и малому размеру цели?

## 2. Экспериментальная матрица

- Архитектуры: `detr_resnet50`, `yolos_tiny`, кастомные модели
- Подмножества данных:
  - `weather`: clear/rain/fog/snow/...
  - `quality`: high/medium/low
  - `maneuver`: straight/turn/rapid_change/...
- Для каждой комбинации: глобальные и condition-wise метрики

## 3. Метрики

- Качество: `mAP`, `mAP50`, `mAP75`, `mAR`
- Эффективность: `latency_ms`, `FPS`
- Интегральная метрика:
  - `Efficiency = mAP50 / latency_ms`

## 4. Протокол обучения

- Фиксированный random seed
- Единый формат данных COCO
- Единые сплиты `train/val/test`
- Одинаковая длительность обучения и критерий выбора лучшего чекпоинта

## 5. Протокол валидации

- Валидация по эпохам на `val`
- Финальная оценка на `test`
- Отдельный анализ по `weather/quality/maneuver`

## 6. Статистическая надежность

- Для ключевых метрик использовать доверительные интервалы bootstrap
- Для попарного сравнения моделей по условиям использовать непараметрические тесты

## 7. Практический вывод

По итогам экспериментов формируется рекомендация:

1. Лучшая модель по точности (server profile)
2. Лучшая модель по скорости (edge profile)
3. Лучшая компромиссная модель по эффективности
